# Visual Data Schema

The v2 JSONL schema remains backward-compatible with the slide-aware schema.

One item is still a `ChallengeItem`. New optional fields:

- `video`: local video path, start/end seconds, sampled frame paths, and FPS.
- `visual_context`: scene summary, OCR text, objects, actions, spatial relations, and frame IDs.
- `reference`: translation text plus provenance and verification status.
- `hard_labels`: evaluation targets for homophones, visual deixis, objects, actions, OCR terms, entities, and terminology.
- `evidence[*]`: can now include modality, visual scope, video/frame id, temporal span, confidence, spoken anchor, and `visual_only`.

References generated by an LLM should use `reference.status=llm_generated`; human-reviewed references should use `human_verified` or `human_edited`.

Local media paths are pointers only. Do not commit videos, sampled frames from real data, model outputs containing private data, or API keys.
